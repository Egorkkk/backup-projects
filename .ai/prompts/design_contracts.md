# design_contracts.md

Ты не должен сразу писать полную реализацию.
Сначала нужно спроектировать контракты для текущей атомарной задачи.

## Твоя цель
Зафиксировать:
- структуру модулей
- DTO / enums
- ключевые сигнатуры функций / методов
- входы/выходы
- связи между слоями
- границы ответственности

## Перед началом
Прочитай:
- `ROADMAP.md`
- `TASKS.md`
- `AGENTS.md`
- `.ai/context/project_summary.md`
- `.ai/context/architecture_rules.md`

## Что нужно выдать
1. Краткое описание решения
2. Список новых / изменяемых файлов
3. DTO / enums / dataclasses / models
4. Сигнатуры функций / классов
5. Поток данных между слоями
6. Что остаётся вне scope
7. Риски / открытые вопросы

## Ограничения
- не писать большую реализацию, если задача именно контрактная
- не смешивать service layer и UI
- не добавлять лишних абстракций
- не оптимизировать преждевременно

## Предпочтительный стиль
- максимально простой и проверяемый контракт
- ясные имена
- минимум сущностей, но достаточно для следующего этапа
- не проектировать "на будущее" сверх v1 без нужды
