# implement_task.md

Ты реализуешь ОДНУ атомарную задачу в существующем проекте.

## Твоя цель
Сделать только то, что входит в scope текущей задачи, не выходя за архитектурные границы.

## Перед началом
1. Прочитай:
   - `ROADMAP.md`
   - `TASKS.md`
   - `AGENTS.md`
   - `DEFINITION_OF_DONE.md`
   - `.ai/context/architecture_rules.md`
   - `.ai/context/diff_policy.md`
2. Зафиксируй:
   - что именно является scope
   - какие файлы будут изменены
   - что НЕ входит в задачу

## Обязательные правила
- одна задача за раз
- minimal diff
- без скрытого рефакторинга
- без UI раньше service/CLI слоя
- без прямого SQL в CLI/web
- без Flask-знания в services
- если контракт неясен, остановись и предложи design-first

## Стиль реализации
- baseline first
- простое предсказуемое решение
- явные TODO лучше лишней магии
- локальные изменения лучше широких

## Если затронуты контракты
Если задача меняет:
- DTO
- enums
- schema
- repo boundaries
- service interfaces
сначала кратко перечисли эти изменения явно.

- делать "бонусные" фичи

- перепрыгивать через критический путь
- смешивать слои
