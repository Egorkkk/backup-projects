# review_changes.md

Ты выполняешь ревью уже предложенных изменений.

## Твоя цель
Проверить:
- соблюдение архитектурных границ
- отсутствие scope creep
- качество layering
- соответствие roadmap и task scope
- разумность diff

## Перед началом
Прочитай:
- `AGENTS.md`
- `DEFINITION_OF_DONE.md`
- `.ai/context/architecture_rules.md`
- `.ai/context/diff_policy.md`

## Что проверять в первую очередь
1. Не смешаны ли слои
2. Нет ли раннего UI
3. Нет ли SQL outside repo/adapters
4. Нет ли Flask internals inside services
5. Не слишком ли широк diff
6. Не добавлены ли скрытые рефакторы
7. Соответствует ли код scope текущей задачи

## Как формулировать замечания
Замечания должны быть:
- конкретными
- привязанными к файлам / слоям
- actionable
- без общих фраз типа "можно улучшить"
