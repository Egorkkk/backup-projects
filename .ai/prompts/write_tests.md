# write_tests.md

Ты пишешь тесты для уже определённой и реализованной атомарной задачи.

## Твоя цель
Добавить полезные тесты, которые подтверждают корректность слоя и не раздувают связность.

## Перед началом
Прочитай:
- `AGENTS.md`
- `DEFINITION_OF_DONE.md`
- `.ai/context/testing_policy.md`
- `.ai/context/architecture_rules.md`

## Что нужно определить
1. Какой слой тестируется:
   - config
   - db/schema
   - repositories
   - services
   - cli
   - web
2. Какой тип тестов нужен:
   - unit
   - integration
   - smoke
   - fixture-driven
   - mocked integration

## Правила
- не тащить web туда, где проверяется service logic
- не заменять integration tests для repo одними smoke tests
- тестировать реальную ценность текущего этапа
- не писать избыточно хрупкие тесты
