# DEFINITION_OF_DONE.md

## Общий DoD для атомарной задачи

Задача считается выполненной, если соблюдены все пункты ниже.

### 1. Scope выполнен
Сделано именно то, что требовалось задачей, без скрытого расширения scope.

### 2. Архитектурные границы не нарушены
- web не знает про sqlite напрямую
- cli не знает про SQL напрямую
- jobs не знают про HTML
- services не знают про Flask request/response

### 3. Контракты согласованы
Если задача затрагивает DTO / schema / service boundaries, они либо:
- были заранее спроектированы,
- либо явно зафиксированы в текущей задаче.

### 4. Код локально проверяем
Есть хотя бы один из вариантов:
- unit tests
- integration tests
- smoke check
- import / startup validation

### 5. Изменения локальны и понятны
Diff обозримый, изменения объяснимые, unrelated files не затронуты без причины.

### 6. Документация обновлена при необходимости
Если меняются:
- структура конфигов
- public CLI behavior
- архитектурные boundaries
- способ запуска
то соответствующие docs должны быть обновлены.

### 7. Есть финальный отчёт
После задачи должен быть отчёт:
- выполнено
- изменённые файлы
- что не делалось намеренно
- проверки
- риски / TODO

---

## Специальные критерии по типу задач

### Для config / constants
- пример конфигурации валиден
- invalid config даёт понятную ошибку
- значения по умолчанию отражены явно

### Для DB/schema/repositories
- schema создаётся
- повторный init обрабатывается корректно
- CRUD/lookup покрыты integration tests где нужно

### Для services
- inputs/outputs ясны
- побочные эффекты ограничены
- логика тестируема без web UI

### Для CLI
- команда вызывается предсказуемо
- help / аргументы не противоречат docs
- команда не содержит бизнес-логики, только orchestration

### Для Flask UI
- route использует service layer
- HTML не содержит бизнес-правил
- без прямого SQL доступа

---

## Anti-DoD

Задача НЕ считается выполненной, если:
- код "вроде написан", но не проверен никак
- нарушены слои
- появился scope creep
- изменения трудно review-ить
- нет отчёта
- нет понятного способа локально проверить результат
